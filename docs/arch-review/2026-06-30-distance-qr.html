Problem

turnpoint.py re-derives perpendicular azimuth and endpoints that GoalLine already owns — the docstring's "single source" claim is false.

Solution

Move the optimal-crossing computation onto GoalLine (it already has endpoints()); have the turnpoint call through it.

  • locality: goal-line rules in one module
  • docstring's claim becomes true
  • one endpoint primitive, one test surface
  • pairs naturally with B
Note: the turnpoint needs a goal line oriented to an arbitrary prev_point, not the task's previous turnpoint. GoalLine currently binds approach to a fixed turnpoint — the deepening must let the approach vary. Worth grilling the interface.

Problem

Six enum pairs are hand-mapped with if/elif in both directions across from_task and to_task — a shallow mapping spread thin.

Solution

One small bidirectional table per enum pair; a tiny convert reads it both ways. Leave the ordering logic alone.

  • locality: each pair defined once
  • add an enum value in one row
  • round-trip becomes table-symmetric
  • scope tight: mapping only, not ordering

Top recommendation

Start with A — collapse the DP machinery.

It is the deepest friction and the most leverage: two hand-synced copies become one parameterized routine, the pass-through that strips goal_type disappears, and the optimizer gains one clear test surface. Do B first as a free warm-up — it is pure deletion — then C follows naturally since B removes one of the two goal-line copies.
